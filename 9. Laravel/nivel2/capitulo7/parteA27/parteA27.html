Compilacion con Vite

Volver al capitulo

Que es?

Desarrollar en Vite es un poco diferente a hacerlo con Webpack, porque Vite no es un paquete. Mas bien es una canalizacion de herramientas preconfiguradas que usan Rollup para la configuracion y un servidor local para el desarrollo.

Para instalar Vite en un nuevo proyecto de Laravel simplemente debemos usar el comando:

npm install

History

Vite is a modern frontend build tool that provides an extremely fast development environment and bundles your code for production. When building applications with Laravel, you will typically use Vite to bundle your application's CSS and JavaScript files into production ready assets.

Laravel integrates seamlessly with Vite by providing an official plugin and Blade directive to load your assets for development and production.

Before transitioning to Vite, new Laravel applications utilized Mix, which is powered by webpack, when bundling assets. Vite focuses on providing a faster and more productive experience when building rich JavaScript applications. If you are developing a Single Page Application (SPA), including those developed with tools like Inertia, Vite will be the perfect fit.

Levantar servidor

Debido a que Vite compila los assets de CSS y JS usando un servidor local, debemos levantarlo en la terminal con el comando:

npm run dev

Hecho esto el servidor se mantendra a la escucha para actualizar cualquier cambio en nuestro codigo CSS y JS (siempre y cuando no lo tumbemos matando la terminal xd), es decir los assets.

Archivo vite.config.js

Este archivo, que se encuentra directamente en el directorio raiz del proyecto, contiene la configuracion de Vite.

El un objeto tenemos la llave "input" que contiene un arreglo. Este arreglo contiene todas las entradas que Vite estara a la escucha para compilar desde la carpeta resourses hasta la carpeta public.

input: [
    'resources/css/app.css',
    'resources/js/app.js',
]

Directiva @vite

Este directiva de Blade permite incluir los archivos compilador en las vistas:

<!-- Scripts -->
@vite(['resources/css/app.css', 'resources/js/app.js'])

Pasar a produccion

Como hemos visto Vite requiere que tengamos un servidor local corriendo cuando estamos en un entorno de desarrollo local. Entonces, como sera cuando los pasemos a produccion?

Cuando este en produccion Vite no esperara que tengamos un servidor corriendo para que compile los assets, lo que espera esque ya tengamos los assets compilador en algun lado para entregarselos al cliente.

Con el objetivo de lograr esto cuando un proyecto este listo para subirlo a produccion, debemos ejecutar el siguiente comando:

npm run build

Este comando va a crear una carpeta build dentro de la carpeta public con los assets. y el archivo manifest.js que sirve para enrutar los assets.

Con esto el proyecto esta listo para ir a produccion!
