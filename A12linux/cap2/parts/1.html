Como funciona

Todos los archivos y directorios en Linux tienen asociado un conjunto de permisos que definen las posibilidades de lectura, escritura y ejecución que se aplican al usuario propietario, al grupo propietario, y al resto del mundo.

Usuarios

Cada usuario se identifica por un User ID o UID. Todos los usuarios en Linux suelen estar registrados en el archivo /etc/passwd. Principalmente hay tres tipos de usuarios:

  • Root/Admin: (UID = 0), que es el único que tiene privilegios sobre todo el sistema, y el responsable de las tareas de administración del sistema (como instalación y desinstalación de software).
  • Usuarios del sistema: usuarios especiales que van vinculados a ciertos servicios, y que pueden asumir ciertos privilegios relativos a este servicio. Se crean automáticamente en la instalación del sistema o de ciertas aplicaciones. Algunos ejemplos de estos usuarios son bin, mail, apache etc.
  • Usuarios estandar: son las cuentas de usuarios individuales. Cada usuario estándar posee su directorio personal dentro de /home. Allí se almacenan todos sus archivos personales, además de las preferencias de varias aplicaciones, archivos temporales, etc.

Grupos

Para simplificar la gestión, Linux agrupa todos los usuarios dentro de grupos. De este modo, a la hora de especificar ciertos permisos sobre un conjunto de cuentas, se pueden establecer de una sola vez sobre el grupo. Cada grupo se identifica por un Group ID o GUID y quedan registrados en el archivo /etc/group.

Cada usuario pertenece por defecto a un grupo homónimo, conocido como "grupo principal de usuario" o "grupo primario". Además, un mismo usuario puede estar dentro de otros grupos, que serían los "grupos secundarios".
