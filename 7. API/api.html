Base de Datos

La BBDD debe cumplir los siguientes requisitos:

La tablas deben estar escritas en plural, y el nombre de las columnas debe terminar con el nombre de la tabla en singular (sufijo):

Tabla categories

  • id_category
  • name_category
  • date_created_category
  • date_updated_category

Las ultimas dos columnas deben ser la fecha de creacion (Tipo DATE) y la fecha de actualizacion (Tipo DATE TIME):

Tabla categories

  • id_category
    • ... Se agregan las columnas nesesarias...
  • date_created_category
  • date_updated_category

Para ralacionar tablas, las Claves Externas deben nombrarse de la siguiente forma: id_(tabla externa en singular)_(tabla actual en singular).

Tabla actual: products
Tabla externa 1: categories | Clave Externa: id_category_product
Tabla externa 2: stores | Clave Externa: id_store_product

Para realizar autenticaciones es indispensable que la tabla de autenticacion tenga siempre estas 4 columnas:

  • email_sufijo
  • password_sufijo
  • token_sufijo
  • token_exp_sufijo
