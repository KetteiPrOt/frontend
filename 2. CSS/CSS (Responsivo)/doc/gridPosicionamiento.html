Estas propiedades sirven para especificar como queremos que se acomode un elemento hijo en las filas dentro de nuestra grid.
Las filas y las columnas se cuentan entre los elementos, no en el centro de ellos.

span (valor)

Es un valor muy util (span=abarcar) de las propiedades grid-column:; y grid-row:; porque sirve para referirse a las filas y las columnas de una forma mucho mas intuitiva y practica.
n = literalmente el numero de filas/columnas que deseamos que abarque dentro de la grid sin tomar en cuenta las confusas coordenadas especificas de la matriz.

grid-column: start / span n; grid-row: start / span n;

Al omitir el primer valor tomara la posicion original del elemento (muy practico).

grid-column: span n; grid-row: span n;

grid-row: span 2; grid-column: span 3; = Ocupa desde tu fila original dos filas mas, y desde tu columna original, tres columnas mas.

Se esta creando automaticamente una nueva fila implicita ajustada a la altura del contenedor padre.
Porque el elemento hijo 12 ya no cabia en la disposicion anterior.

grid-template-areas

Con grid area podemos separar un espacio dentro del contenedor padre (un area) y nombrarlo con una etiqueta. Esto es para asignarle ese espacio a un elemento usando la etiqueta y sin preocuparnos por los molestos numeros. Esta propiedad permite crear una de esas areas dentro del contenedor padre y nombrarla con una etiqueta.

Area de 1 x 1 ['label']. Ocupa la fila uno y columna uno. Recibe el nombre de areaUno.

grid-template-areas: 'areaUno'; Activa la visualizacion de Grid en DevTools para ver el area y su etiqueta.

areaUno asignada al elemento hijo 1 [row: areaUno/areaUno; column: areaUno/areaUno;]
