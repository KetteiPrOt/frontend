__proto__ (Dunder proto)

Es una propiedad de los objetos que contiene el prototipo con el que fueron creados. Por ende, todo objeto hereda las propiedades y metodos de su prototipo contenido en esta propiedad.

class Clase { constructor() {} };

let texto = 'Hola',
    numero = 23,
    booleano = true,
    funcion = function() {},
    objeto = { propiedad: 'valor' },
    array = ['pedro', 'juan', 'burro'];

console.log(texto.__proto__); // String { ... }
console.log(numero.__proto__); // Number { ... }
console.log(booleano.__proto__); // Boolean { ... }
console.log(funcion.__proto__); // ƒ () { [native code] }
console.log(objeto.__proto__); // { constructor: ƒ Object() ... }
 console.log(Clase.__proto__); // ƒ () { [native code] }
console.log(array.__proto__); // { constructor: ƒ Array() ... }

prototype

Es una propiedad de los objetos que tiene el prototipo actual del objeto, es decir el prototipo es un plantilla, entonces si un objeto tiene definida una plantilla para crear mas objetos (por ejemplo una funcion contructora, o una clase)... la propiedad prototype contendra esta plantilla.

class Clase { constructor() {} };

let primitivo = 44,
    funcionConstructora = function() {};

console.log(primitivo.prototype); // undefined
console.log(funcionConstructora.prototype); // { constructor: ƒ () ... }
console.log(Clase.prototype); // { constructor: class Clase ... }

Clases y prototipos

Crear una clase es crear un prototipo... a su vez los objetos que son creados usando esa clase heredan el prototipo creado con ella.

class Clase {
    constructor(valor) {
         this.propiedad = valor;
    }
    metodo() {}
}

const objeto = new Clase('valor');

console.log(objeto.__proto__); // { constructor: class Clase, metodo: ƒ metodo() ... }

Modificar prototipo

En esta programacion los Prototipos crean una conexion entre los objetos y su Clase... ya que es posible alterar la clase (el prototipo) y los futuros elementos creados con ella, accediendo a la clase/prototipo desde cualquier objeto. Esto en otros lenguajes basados en clases es muy extranio.

class Objetos {
    constructor(valor1, valor2, valor3) {
         this.propiedad1 = valor1;
         this.propiedad2 = valor2;
         this.propiedad3 = valor3;
    }
    metodo1() { console.log('Ejecutado: Metodo 1') }
    metodo2() { console.log('Ejecutado: Metodo 2') }
    metodo3() { console.log('Ejecutado: Metodo 3') }
}

const objeto1 = new Objetos(1, 2, 3);

objeto1.metodo1(); // Ejecutado: Metodo 1

objeto1.__proto__.metodo1 = () => { console.log('Metodo modificado') }

const objeto2 = new Objetos(1, 2, 3);

objeto2.metodo1(); // Metodo modificado
