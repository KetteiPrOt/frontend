Modo estricto

Volver al capitulo

'use strict';

Esta cadena de texto de usa para activar el modo estricto, obligatoriamente tiene que ir al inicio del bloque. Este modo convierte algunos errores en aprentemente triviales en excepciones para tener un codigo mas optimo y veloz.

Variables declaradas

Al usar el modo estricto se vuelve obligatorio declarar las variables.

'use strict';

nombre = 'Joel'; // (!) Error

Propiedades de solo lectura

Al intentar escribir propiedades de solo lectura en modo estricto recibiremos un error.

'use strict';

const perro = {};

Object.defineProperty(perro, 'color', { value: 'rojo', writeable: false });

perro.color = 'cafe'; // (!) Error

console.log(perro);

Objetos restringidos

Al intentar extender objetos restringidos por preventExtensions recibiremos un error.

'use strict';

const objeto = {};

Object.preventExtensions(objeto);

objeto.propiedad = 'DINOSAURIO'; // (!) Error

Sin propiedades en datos primitivos

El modo estricto prohibe agregar propiedades a datos primitivos lanzando una excepcion. En modo normal este intento no tendria ningun efecto.

'use strict';

let texto = 'Joel';

texto.pais = 'Ecuador'; // (!) Error

No duplicar parametros de una funcion

El modo estricto prohibe duplicar nombres de argumentos de funciones lanzando un error. En modo normal la funcion solo usaria el ultimo parametro escrito.

'use strict';

const funcion = (arg1, arg2, arg1) => {} // (!) Error

Solo eliminar propiedades

En el modo estricto intentar eliminiar alguna otra cosa que no sea una propiedad de un objeto con la sentencia delete arroja un error.

'use strict';

let variable = 'Hola';

delete variable; // (!) Error
