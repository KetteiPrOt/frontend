Sentencia de bloque

Las sentencia de bloque se define con {}, y definen un nuevo ambito de variables.

let nombre = 'pedro';
{
    let nombre = 'jorge';
    alert(nombre); // jorge
}
alert(nombre); // pedro

Errores y sus tipos basicos

Son errores inesperados al ejecutar el programa. Los errores fundamentales que podemos obtener en JavaScript son:

  • Error
  • EvalError
  • InternalError
  • RangeError
  • ReferenceError
  • SintaxError
  • TypeError
  • URIError

try { ... } catch(error) { ... }

Cuando usamos try se desactivan los Event Handlers (Manipuladores de eventos) que disparan los errores. En su lugar si hay un error este es pasado como parametro a catch, para asi poder manejarlo nosotros.

Esta sentencia no controla los errores de sintaxis, sino mas bien los errores logicos en el programa que podriamos esperar.

try {
    variable
} catch(error) {
    console.log(error); // ReferenceError: variable is not defined
console.log(typeof error); // Object
}

finally

Esta sentencia define un bloque de codigo que se ejecutara haya, o no haya ocurrido un error.

try {
    let variable = 45;
} catch {
    console.log('Ha ocurrido un error en el primer codigo');
} finally {
    console.log('La primera prueba ha finalizado'); // La primera prueba ha finalizado
}

try {
    variable
} catch {
    console.log('Ha ocurrido un error en el segundo codigo'); // Ha ocurrido un error en el segundo codigo
} finally {
    console.log('La segunda prueba ha finalizado'); // La segunda prueba ha finalizado
}

throw

Esta sentencia lanza lo que sea que le pasemos como un error.

try {
    throw 'Guitarra, soledad y un planeta azul';
} catch (error) {
    console.log(error); // Guitarra, soledad y un planeta azul
}
