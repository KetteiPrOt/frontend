Definicion y usos

Una funcion es un bloque del programa quien recibe uno o varios valores llamados argumentos y retorna un unico valor de salida al finalizar sus procesos internos. Son estructuras muy utiles para no repetir codigo en programas que requieran realizar un proceso mismo proceso en mas de una ocasion, y tambien para tener el codigo mas estructurado y ordenado dentro del programa.

Formas de crear e invocar funciones

Hay dos formas de crear una funcion, una de ellas es:

function nombre (argumento1, argumento2...) {
  // codigo
}

La otra es la siguiente:

const nombre = function (argumento1, argumento2...) {
  // codigo
}

Y para llamarla a ejecutar su codigo se la invoca de la siguiente manera sin importar la forma en que fue creada:

nombre('valor1', 'valor2'...);

Retorno de la funcion (return)

Las funciones reciben varios argumentos de entrada y retornan un unico valor de salida. Este valor se especifica con la sentencia return:

function sumar (numero1, numero2) {
  let suma = numero1 + numero2;
  return suma;
}

De esta forma, al invocarla y pasarle los argumentos la funcion devolvera el valor de la variable suma:

let resultado = sumar(5, 18);

console.log(resultado); // 23

Tambien es importante saber que al ejecutar la sentencia return la funcion finaliza su ejecucion de manera similar a cuando un bucle se para por la sentencia break.
Ya que ha cumplido su mision de retornar su valor de salida se puede ir en paz.

Declaracion de argumentos

Definir un argumento al crear una funcion es equivalente a declarar una variable sin inicializarla, por lo que:

function (argumento1, argumento2) { ... };

Es equivalente a:

let argumento1, argumento2;

Y los valores de argumento1 y argumento2, en ambas situaciones, son indefinidos (undefined) ya que no han sido inicializados.

Funcion flecha

Una funcion flecha es una alternativa compacta a una funcion normal, con algunas limitaciones especiales . Cuando son funciones de una sola linea se puede abreviar bastante:

// Funcion Normal Escrita de la forma 1
function sumarCentena (a) {
  return 100 + a;
}
console.log(sumarCentena(7)); // 107

// Funcion Normal Escrita de la forma 2
const sumarCentena = function (a) {
  return 100 + a;
}
console.log(sumarCentena(7)); // 107

// Funcion Flecha
const sumarCentena = a => a + 100;
console.log(sumarCentena(7)); // 107

Las Arrow Functions (Funciones Flecha) no son recomendadas para ser usadas como metodos de objetos y no se pueden usar como funciones contructoras.

Cuando la funcion tiene mas argumentos y/o mas lineas de codigo se vuelve nesesario utilizar signos de agrupacion (parentesis y corchetes), asi como especificarle el retorno (return) que antes era implicito:

// Funcion Normal
function nombre (argumento1, argumento2...) {
  // codigo
  return valorRetorno;
}
// Funcion Flecha
const nombre = (argumento1, argumento2...) => {
  // codigo
  return valorRetorno
}
