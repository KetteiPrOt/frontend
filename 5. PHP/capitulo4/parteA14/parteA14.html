Que es una interfaz

Aveces es nesesario que un grupo de personas trabajen juntas. En este caso es imprescindible crear unas pautas generales de trabajo para que el resultado final sea el esperado. Si el desarrollo consiste en programar varios objetos, el analista de la aplicacion puede definir la estructura basica en papel o crear pequenia plantilla con metodos que el objeto final deberia tener obligatoriamente. Esta plantilla se le conoce como interfaz, y permite establecer una clase con fucniones definidas pero sin desarrollar. Esta plantilla obliga a todas las clases que la implementen declarar estos metodos, como minimo.

En pocas palabras, las interfacez son plantillas.

Crear una interfaz

Usamos la palabra clave inerface y a continuacion definimos los metodos que deberian tener las clases que implementan esta interfaz. Estos metodos solo pueden ser publicos.

interface iTemplate {
    public function setVariable();
    public function getHTML();
}

Implementar una interfaz

Para implementar una interfaz en una clase usamos implements. Si la clase creada no tiene los metodos escritos en la interfaz va a dar un error.

interface iTemplate {
    public function setVariable($dato);
    public function getHTML();
}

class Template implements iTemplate {
    // (!) Error: La clase Template tiene dos metodos abstractos
}

Si declaramos los metodos el error desaparece.

interface iTemplate {
    public function setVariable($dato);
    public function getHTML();
}

class Template implements iTemplate {
    public function setVariable($dato) {
         //
    }

    public function getHTML() {
         //
    }
}

Implementar Varias Interfacez

Una clase puede implementar varias interfacez.

interface A {
    public function prueba1();
}

interface B{
    public function prueba2();
}

class Testing implements A, B {
    public function prueba1()
    {
         //
    }

    public function prueba2()
    {
         //
    }
}

Extender una interfaz

Es posible extender una interfaz al igual que una clase usando extends. Aqui la clase Testing se vera obligada a declarar los metodos prueba1 y prueba2, ya que la interfaz B es una extension de la interfaz A.

interface A {
    public function prueba1();
}

interface B extends A {
    public function prueba2();
}

class Testing implements B {
    public function prueba1()
    {
         //
    }

    public function prueba2()
    {
         //
    }
}

Las interfacez admiten multiherencia.

interface A {
    public function prueba1();
}

interface B {
    public function prueba2();
}

interface C extends A, B {
    public function prueba1();
}

class Testing implements C {
    public function prueba1()
    {
         //
    }

    public function prueba2()
    {
         //
    }

    public function prueba3()
    {
         //
    }
}

Clases Abstractas

Su funcion es similar a las interfacez. Pero a diferencia de ellas en las clases abstractas podemos desarrollar los metodos que definamos y tambien podemos definirlos como privados o protegidos, cosa que no era posible en las interfacez.

Para crear una clase abstracta y sus metodos usamos abstract. Todas las clases que extiendan esta clase deberan definir los metodos abstractos (al igual que en una interfaz) escritos en ella, publicos, privados, o protegidos.

# Clase Abstracta
abstract class ClaseAbstracta {
    abstract protected function getValor(); // Metodo Abstracto

    public function setValor($dato)
    {
         // Metodo normal
    }
}

Usar una clase abstracta

Una clase abstracta no puede ser instanciada. Para poder usar una clase abstracta es nesesario primero que otra clase la extienda.

# Clase Abstracta
abstract class ClaseAbstracta {
    abstract protected function getValor();

    public function mostrarValor()
    {
         echo $this->getValor();
    }
}

class ClaseConcreta extends ClaseAbstracta {
    public function getValor(): string
    {
         return "Dato";
    }
}

$objeto = new ClaseConcreta;

$objeto->mostrarValor(); // Dato
